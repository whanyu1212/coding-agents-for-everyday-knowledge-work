# Coding Agents for Everyday Knowledge Work

An Astro slide deck for a technical sharing session on coding agents, OpenCode,
runtime harnesses, context economics, and agent workflows that go beyond code
generation.

Live site: https://whanyu1212.github.io/coding-agents-for-everyday-knowledge-work/

## What The Talk Covers

- Coding agents as systems that act through code, not just code generators.
- OpenCode runtime anatomy: prompts, sessions, tools, events, and continuation.
- Extension points: skills, plugins, hooks, custom tools, and MCPs.
- Oh-My-OpenAgent as an opinionated harness layer on top of OpenCode.
- RTK and related tools for reducing token waste from command output and context bloat.
- Agent-as-optimizer workflows: measurable loops, scoring, gates, and iteration.
- The core thesis: own the smallest harness you can understand, debug, and change.

## Local Development

Requirements:

- Node.js `>=22.12.0`
- npm

Install dependencies:

```sh
npm install
```

Start the dev server:

```sh
npm run dev
```

Build the static site:

```sh
npm run build
```

Preview the production build locally:

```sh
npm run preview
```

## Project Structure

```text
.
|-- src/
|   |-- components/       # Slide, nav, card, and code-block components
|   |-- layouts/          # Global HTML shell and CSS variables
|   `-- pages/index.astro # Main slide deck content and slide-specific styles
|-- public/               # Static assets and samples
|-- refs/                 # Local reference snapshots used while preparing the talk
|-- astro.config.mjs      # GitHub Pages site/base config
`-- .github/workflows/    # Build and GitHub Pages deployment workflow
```

Most content edits happen in `src/pages/index.astro`. The deck is intentionally
self-contained: slide markup and slide-specific CSS live together so the talk can
be iterated quickly.

## Deployment

The project is configured for GitHub Pages under the repository subpath:

```js
site: 'https://whanyu1212.github.io'
base: '/coding-agents-for-everyday-knowledge-work/'
```

The workflow in `.github/workflows/deploy.yml` builds on pull requests to
`main`, and deploys only on pushes to `main`.

## Notes

- `refs/` contains source and documentation snapshots used to ground claims in
  the deck. It is not required for normal Astro development.
- `dist/` is generated by `npm run build` and should not be edited directly.
- The deck currently builds as a static Astro site.
